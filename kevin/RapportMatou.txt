Features / Implémentation :
	- On ne met pas à jour OP_ACCEPT car il faut pouvoir accepter une connexion pour la refuser
	- Un seul bytebuffer d'écriture
	- Si le fichier "files" a disparu...


	
Disclaimer :
	- Colorator
	- SelectorLogger
	
Si rien n'est précisé : ByteBuffer en write mode
Commandes : exit, etc

Points forts :
	- Très peu d'allocations côté serveur
	- Un ClientCore peut créer plusieurs ClientInstance
	- Ne dépend pas d'une seule interface utilisateur
	- ThreadGroup
	- Pseudo ignorant la casse

Points faibles :
	- Architecture compliquée (Core + Instance/Database + Session)
	- Systèmes d'évenements...
	
Difficultés rencontrés :
	- ByteBuffers...
	- Gestion des évenements (inspiré de Visiteur)

Bug connu :
	- Localhost privé
	- Le Sender Thread n'est pas interrompu à cause du Shell
	- Pseudo et alphabet bizarre (Georgian)
	- SOURCE accepte DESTINATION si même IP
	
Améliorations possibles :
	- Déconnecter les inactifs non authentifiés depuis plus d'1 minute.
	- Déconnecter les inactifs authentifiés depuis plus de 10 minutes.
	- Pouvoir refuser des fichiers
	- Annuler une demande /open
